## Consistent Architecture Across Templates

Every template follows the same proven architecture, making it easy to switch between stacks or apply patterns across projects:

### Feature-Based Organization

All templates use a scalable, feature-based folder structure:

<Files>
  <Folder name="src" defaultOpen>
    <Folder name="features">
      <Folder name="auth">
        <File name="auth.controller.ts" />
        <File name="auth.procedure.ts" />
        <File name="auth.schema.ts" />
      </Folder>
      <Folder name="users">
        <File name="users.controller.ts" />
        <File name="users.procedure.ts" />
        <File name="users.schema.ts" />
      </Folder>
    </Folder>
    <Folder name="lib">
      <File name="database.ts" />
      <File name="logger.ts" />
    </Folder>
    <File name="igniter.ts" />
    <File name="igniter.router.ts" />
    <File name="igniter.client.ts" />
  </Folder>
</Files>

### Shared Conventions

**Controller Pattern**: Type-safe API endpoints with automatic validation

```typescript
export const userController = igniter.controller({
  path: '/users',
  actions: {
    list: igniter.query({
      path: '/',
      handler: async ({ context, response }) => {
        const users = await context.db.user.findMany()
        return response.success({ users })
      }
    })
  }
})
```

**Procedure Pattern**: Reusable business logic and middleware

```typescript
export const auth = igniter.procedure({
  handler: async (options, { context, response }) => {
    const user = await getCurrentUser(context)
    if (options.required && !user) {
      return response.unauthorized()
    }
    return { auth: { user } }
  }
})
```

**Schema Pattern**: Centralized validation with Zod

```typescript
export const createUserSchema = z.object({
  name: z.string().min(1),
  email: z.string().email()
})
```

## Getting Started: 2-Minute Setup

Every template includes comprehensive setup instructions, but the basics are the same:

<Steps>
  <Step>
    ### Initialize Project
    
    <Tabs items={['Interactive', 'Direct']} groupId="init-method">
      <Tab value="Interactive">
        ```bash
        npx @igniter-js/cli init my-app
        # Follow prompts to select template
        ```
      </Tab>
      
      <Tab value="Direct">
        ```bash
        npx @igniter-js/cli init my-app --template nextjs
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step>
    ### Install Dependencies
    
    <Tabs items={['npm', 'pnpm', 'yarn', 'bun']} groupId="package-manager">
      <Tab value="npm">
        ```bash
        cd my-app
        npm install
        ```
      </Tab>
      
      <Tab value="pnpm">
        ```bash
        cd my-app
        pnpm install
        ```
      </Tab>
      
      <Tab value="yarn">
        ```bash
        cd my-app
        yarn install
        ```
      </Tab>
      
      <Tab value="bun">
        ```bash
        cd my-app
        bun install
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step>
    ### Start Local Services
    
    ```bash
    # Start PostgreSQL and Redis with Docker
    docker-compose up -d
    ```
  </Step>

  <Step>
    ### Configure Environment
    
    ```bash
    # Copy example environment file
    cp .env.example .env
    
    # Edit .env with your settings
    # DATABASE_URL and REDIS_URL are pre-configured for Docker
    ```
  </Step>

  <Step>
    ### Initialize Database
    
    ```bash
    npx prisma db push
    ```
  </Step>

  <Step>
    ### Start Development Server
    
    ```bash
    npm run dev
    ```
    
    Your application is now running at `http://localhost:3000`
  </Step>
</Steps>
